# E-Kanban 开发任务：2.2 实现工作项同步服务

## 任务说明
按照 E-Kanban 开发计划，第二批次第二个任务：实现从 Azure Boards 同步工作项到 E-Kanban 的服务。

## 前置条件
- ✅ 2.1 Azure Boards API 客户端已完成

## 开发规范
$(cat docs/volcore-development-guide.md)

## 位置
- 接口：`EKanban/IServices/ISyncService.cs`
- 实现：`EKanban/Services/SyncService.cs`
- 自定义业务放在 `EKanban/Services/Partial/`

## 功能要求
1. 同步入口方法：
   ```csharp
   Task<SyncResult> SyncFromAzureBoardsAsync();
   ```

2. 同步逻辑：
   - 从 Azure Boards 获取所有工作项
   - 对比本地数据库已存在的工作项
   - 新增或更新本地 `BoardWorkItem`
   - 自动创建对应 `ExecutionCard`
   - 设置状态为 New
   - 返回同步结果（新增多少，更新多少）

3. 幂等处理：
   - 同一个 Azure 工作项只会创建一次 `ExecutionCard`
   - 重复同步只更新标题/描述等信息

## 依赖
- `IAzureBoardsClient` - Azure API 客户端
- `IBoardWorkItemRepository` - 仓储

## 输出要求
1. 创建接口和实现文件
2. 实现完整的同步逻辑
3. 添加适当的日志记录
4. 返回清晰的同步结果
5. 在 `implementation-plan.md` 中更新此任务状态为 ✅ 完成
