# E-Kanban 开发任务：2.3 实现 Spec Engine（Spec 定义 + 评估）

## 任务说明
按照 E-Kanban 开发计划，第二批次第三个任务：实现 Spec Engine，包括 Spec 生成和 Spec 评估。

## 前置条件
- ✅ 第一批次基础设施已完成

## 位置
- 放在 `EKanban/Specs/` 目录
- 生成接口：`ISpecGenerator.cs`
- 生成实现：`SpecGenerator.cs`
- 评估接口：`ISpecEvaluator.cs`
- 评估实现：`SpecEvaluator.cs`

## 功能要求

### ISpecGenerator - Spec 生成
```csharp
Task<Spec> GenerateSpecAsync(ExecutionCard card);
```
- 使用 GitHub Copilot CLI 根据卡片描述自动生成 Spec
- Spec 定义了任务完成的验收标准
- 保存到数据库，关联到 ExecutionCard
- 返回生成的 Spec

### ISpecEvaluator - Spec 评估
```csharp
Task<SpecEvaluation> EvaluateAsync(Spec spec, ExecutionRun run);
```
- 使用 GitHub Copilot CLI 根据 Spec 评估提交的证据
- 判断是否符合 Spec 定义的完成标准
- 返回评估结果（通过/不通过）+ 评估理由
- 保存评估记录到数据库

## 依赖
- `ICopilotCliClient` - GitHub Copilot CLI 客户端（后续 3.1 创建，但这里可以先依赖接口）
- `ISpecRepository` - Spec 仓储
- `ISpecEvaluationRepository` - 评估记录仓储

## Prompt 模板建议

### 生成 Spec 的 prompt：
```
请根据以下工作项描述，生成一个清晰可验证的验收标准 Spec。
工作项标题：{{title}}
工作项描述：{{description}}

请按以下格式输出 Spec：
1. 功能描述
2. 验收标准（每条都要可验证）
3. 输出要求
4. 约束条件
```

### 评估 Spec 的 prompt：
```
请根据给定的 Spec 和提交的执行证据，评估任务是否完成。

Spec (验收标准):
{{spec}}

提交的证据:
{{evidence}}

请评估：是否满足所有验收标准？
输出格式：
RESULT: [PASS/FAIL]
REASONING: 
<你的评估推理过程>
```
