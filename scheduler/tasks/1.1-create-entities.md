# E-Kanban 开发任务：1.1 创建 E-Kanban 领域实体

## 任务说明
按照 E-Kanban 开发计划，第一批次第一个任务：创建 6 个 E-Kanban 领域实体。

## 开发规范
$(cat docs/volcore-development-guide.md)

## 需求
$(cat docs/requirements-structured.md)

## 位置
所有实体放在 `VOL.Entity/DomainModels/` 目录下。

## 需要创建的实体列表

### 1. BoardWorkItem.cs (Azure Boards 工作项)
字段：
- Id (int, 主键)
- AzureWorkItemId (int, Azure Boards 工作项 ID)
- Title (string, 标题)
- Description (string, 描述)
- State (string, 原始状态)
- AreaPath (string, 区域路径)
- IterationPath (string, 迭代路径)
- CreatedDate (DateTime)
- UpdatedDate (DateTime)
- CreatedBy (string)
- ChangedBy (string)

### 2. ExecutionCard.cs (Kanban 执行卡片)
字段：
- Id (int, 主键)
- BoardWorkItemId (int, 关联 BoardWorkItem)
- Title (string, 标题)
- Description (string, 描述)
- Status (int, 状态：0=New, 1=Ready, 2=InProgress, 3=Submitted, 4=Completed, 5=Failed)
- ExecutorType (string, 执行者类型：Human/AI/System)
- ExecutorId (string, 执行者 ID)
- CurrentSpecId (int, 当前 Spec ID)
- LastStartedAt (DateTime, 最后一次开始执行时间)
- LastSubmittedAt (DateTime, 最后一次提交时间)
- FailureCount (int, 失败次数，默认 0)
- NeedsManualIntervention (bool, 是否需要人工干预，默认 false)
- CreatedAt (DateTime)
- UpdatedAt (DateTime)

### 3. ExecutionTask.cs (执行任务定义)
字段：
- Id (int, 主键)
- ExecutionCardId (int, 关联 ExecutionCard)
- TaskPrompt (string, 任务提示词)
- ExpectedOutput (string, 期望输出描述)
- Order (int, 执行顺序)
- IsCompleted (bool)
- CompletedAt (DateTime?)

### 4. ExecutionRun.cs (执行记录，每次执行一个事实)
字段：
- Id (int, 主键)
- ExecutionCardId (int, 关联 ExecutionCard)
- ExecutionTaskId (int, 关联 ExecutionTask，可为 null)
- RunNumber (int, 执行轮次)
- ExecutorType (string, 执行者类型)
- ExecutorId (string, 执行者 ID)
- PromptSent (string, 发送的提示词)
- OutputReceived (string, 收到的输出)
- EvidencePaths (string, 证据文件路径，JSON 数组)
- StartedAt (DateTime)
- CompletedAt (DateTime)
- IsSuccess (bool, 执行是否成功)
- ErrorMessage (string, 错误信息)

### 5. Spec.cs (Spec 定义)
字段：
- Id (int, 主键)
- ExecutionCardId (int, 关联 ExecutionCard)
- SpecContent (string, Spec 内容，定义完成标准)
- GeneratedAt (DateTime)
- GeneratedBy (string, 生成者)
- Version (int, 版本号)

### 6. SpecEvaluation.cs (Spec 评估记录)
字段：
- Id (int, 主键)
- SpecId (int, 关联 Spec)
- ExecutionRunId (int, 关联 ExecutionRun)
- EvaluationResult (bool, 是否通过)
- EvaluationReasoning (string, 评估推理过程)
- EvaluatedAt (DateTime)
- EvaluatedBy (string, 评估者)

## 代码风格要求
- 继承 `VOL.Entity.SystemModels.BaseEntity`
- 使用 SqlSugar 特性标注主键 `[SugarColumn(IsPrimaryKey = true, IsIdentity = true)]`
- 添加 `[Entity(TableCnName = "XXX", TableName = "XXXX")]` 特性
- 遵循 VOL.Core 现有的注释风格
- 使用适当的可空类型引用 (nullable reference types)
- 命名采用 PascalCase

## 输出要求
1. 在 `VOL.Entity/DomainModels/` 目录下创建 6 个文件
2. 每个实体一个文件，文件名就是实体名
3. 确保语法正确，可以编译通过
4. 创建完成后，在 implementation-plan.md 中更新此任务状态为 ✅ 完成
