### 3.3 IAiTaskCheckService - AI 任务超时检查
```csharp
// 检查所有超时的 InProgress AI 任务
Task<CheckResult> CheckInProgressTasksAsync();
```
- 扫描所有 Status = InProgress 且 ExecutorType = AI 的卡片
- 检查 `LastStartedAt` 是否超过配置的超时时间（默认 2 小时）
- 对于超时任务：
  - 如果失败次数 < 最大重试次数 → 自动触发重新执行
  - 如果失败次数 >= 最大重试次数 → 标记为 NeedsManualIntervention，通知管理员
- 返回检查结果（发现多少超时，触发多少重试）

### 3.3 AiTaskCheckJob - Quartz 定时检查任务
- 调用 `IAiTaskCheckService.CheckInProgressTasksAsync()`
- 记录检查结果日志

### 3.4 重新执行触发逻辑
在 `IExecutionCardService` 添加方法：
```csharp
// 手动触发重新执行
Task<Result> TriggerReExecuteAsync(int cardId);
```
- 将卡片状态从 InProgress/Failed 转为 Ready
- 重置状态，准备下一轮执行
- 可以被手动触发（API）或自动触发（超时检查）

## 配置项（在 appsettings.json 中新增）
```json
"AiExecution": {
  "InProgressTimeoutMinutes": 120,
  "MaxAutoRetries": 3,
  "CheckIntervalMinutes": 15
}
```

## 防偷懒机制工作流程
1. AiTaskCheckJob 每 15 分钟运行一次
2. 扫描所有 InProgress 状态的 AI 任务
3. 如果超过超时时间（默认 2 小时）没有提交 → 判定为超时（AI 偷懒了）
4. 如果失败次数 < 最大重试次数 → 自动触发重新执行
5. 如果超过最大重试次数 → 标记为需要人工干预，通知管理员
6. 管理员可以手动触发重新执行
