# E-Kanban 开发任务：2.5 实现 Submit API 核心逻辑

## 任务说明
按照 E-Kanban 开发计划，第二批次第五个任务：实现统一 Submit API 核心逻辑，所有执行结果都通过这个入口提交。

## 前置条件
- ✅ 2.3 Spec Engine 已完成
- ✅ 2.4 状态机已完成

## 开发规范
$(cat docs/volcore-development-guide.md)

## 位置
- 接口：`EKanban/IServices/ISubmitService.cs`
- 实现：`EKanban/Services/SubmitService.cs`

## 功能要求
统一提交入口，支持人/AI/系统三种执行者提交：

```csharp
// 提交执行结果
Task<Result<ExecutionRun>> SubmitAsync(int cardId, string evidence, string executorId);

// 重新提交（重新执行后提交）
Task<Result<ExecutionRun>> ResubmitAsync(int cardId, string evidence, string executorId);
```

## 工作流程
1. 接收提交请求
2. 创建 `ExecutionRun` 记录执行事实
   - 记录执行者、证据、时间
3. 调用状态机流转到 Submitted 状态
4. 触发 Spec 评估
   - 根据当前 Spec 评估证据
5. 根据评估结果：
   - 如果评估通过 → 流转到 Completed 状态
   - 如果评估不通过 → 流转到 Ready 状态，失败次数 +1
6. 如果评估失败且失败次数超过限制 → 标记为需要人工干预
7. 回写结果到 Azure Boards（如果配置了）
8. 返回最终结果

## 依赖
- `IExecutionCardService` - 卡片服务
- `IExecutionRunService` - 执行记录服务
- `IStateMachineService` - 状态机服务
- `ISpecEvaluator` - Spec 评估器
