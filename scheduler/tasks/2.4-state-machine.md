## 状态定义
```csharp
public enum ExecutionCardStatus
{
    New = 0,        // 新建，等待生成 Spec
    Ready = 1,      // Ready，等待执行
    InProgress = 2, // 执行中
    Submitted = 3,  // 已提交，等待评估
    Completed = 4,  // 已完成
    Failed = 5      // 失败，需要干预
}
```

## 允许的状态流转
| 当前状态 \ 目标状态 | New | Ready | InProgress | Submitted | Completed | Failed |
|-------------------|-----|-------|------------|-----------|-----------|--------|
| New               | -   | ✅ | ❌ | ❌ | ❌ | ❌ |
| Ready             | -   | - | ✅ | ❌ | ❌ | ✅ |
| InProgress        | -   | - | - | ✅ | ❌ | ✅ |
| Submitted         | -   | - | - | - | ✅ | ✅ |
| Completed         | -   | - | - | - | - | - |
| Failed            | -   | ✅ | - | - | - | - |

## 需要实现的方法
```csharp
// 检查是否允许流转
bool CanTransition(ExecutionCard card, ExecutionCardStatus newStatus);

// 执行状态流转
Task<Result> TransitionToAsync(ExecutionCard card, ExecutionCardStatus newStatus);

// 流转到 Ready 并生成 Spec
Task<Result> TransitionToReadyAndGenerateSpecAsync(ExecutionCard card);

// 开始执行（流转到 InProgress）
Task<Result> StartExecutionAsync(ExecutionCard card);
```

## 业务规则
- New → Ready：自动生成 Spec 后才能进入 Ready
- Ready → InProgress：记录开始时间，重置失败计数
- InProgress → Submitted：提交后等待 Spec 评估
- Submitted → Completed：评估通过，完成
- Submitted → Failed / Ready：评估不通过，回到 Ready 等待下一轮，或者标记失败
- Failed → Ready：人工干预后重新开始
