# 看板前后端对应关系检查需求分析

## 文档信息
- **版本**：1.0.0
- **作者**：系统开发团队
- **更新时间**：2026-04-07
- **状态**：初稿

## 功能概述

本需求旨在检查前端 `/kanban/index` 页面的功能实现是否与后端 API 接口完全对应，确保系统的一致性和稳定性。主要包括：

- 检查前端同步 Azure Boards 功能与后端 API 的对应关系
- 检查前端卡片详情查看功能与后端 API 的对应关系
- 检查前端数据加载与后端 API 的对应关系
- 检查前端执行触发功能与后端 API 的对应关系

## 前端功能分析

### 1. 页面结构

前端 `/kanban/index` 页面（`src/frontend/vol.web/src/views/Kanban/Index.vue`）主要包含以下功能：

- 看板卡片展示（按状态分组）
- 同步 Azure Boards 功能
- 卡片详情查看功能
- 触发执行和重新执行功能

### 2. API 调用分析

前端通过 `src/frontend/vol.web/src/api/kanban.ts` 中定义的方法与后端交互，主要调用以下 API：

- `getKanbanData()` - 获取看板数据
- `getExecutionCardDetail(id)` - 获取卡片详情
- `triggerSyncFromAzureBoards()` - 触发从 Azure Boards 同步
- `triggerAiExecution(id)` - 触发 AI 执行
- `triggerReExecute(id)` - 触发重新执行

## 后端 API 分析

### 1. 控制器分析

后端主要通过以下控制器提供 API 接口：

- `ExecutionCardController` - 提供卡片相关操作
- `AzureBoardsSyncController` - 提供 Azure Boards 同步功能
- `AiExecutionController` - 提供 AI 执行相关功能

### 2. API 接口分析

后端提供的主要 API 接口：

- `GET /api/ekanban/ExecutionCard/GetKanbanData` - 获取看板数据
- `POST /api/ekanban/AzureBoardsSync/TriggerSync` - 触发 Azure Boards 同步
- `POST /api/ekanban/AiExecution/TriggerExecution` - 触发 AI 执行
- `POST /api/ekanban/ExecutionCard/TriggerReExecute` - 触发重新执行

## 问题分析

通过对比前端调用和后端实现，发现以下问题：

### 1. 详情接口不匹配

- 前端调用 `getExecutionCardDetail(id)` 方法，请求 `/api/ekanban/ExecutionCard/GetDetail` 接口
- 后端 `ExecutionCardController` 中没有实现 `GetDetail` 方法，只有 `GetById` 方法

### 2. 数据结构转换

- 前端对后端返回的数据进行了结构转换，确保与前端组件所需的数据结构一致
- 后端返回的数据结构与前端期望的结构存在差异，需要通过转换来适配

## 解决方案

### 1. 后端添加 GetDetail 接口

在 `ExecutionCardController` 中添加 `GetDetail` 方法，实现与前端调用的对应：

```csharp
[HttpGet]
public async Task<IActionResult> GetDetail(int id)
{
    // 实现获取卡片详情的逻辑，包括阶段进度和文件变更
    // 返回与前端期望结构一致的数据
}
```

### 2. 确保数据结构一致性

确保后端返回的数据结构与前端期望的结构一致，或者在前端进行适当的数据转换，以确保系统的正常运行。

## 验证计划

1. 检查后端是否实现了所有前端调用的 API 接口
2. 验证 API 接口返回的数据结构是否与前端期望的结构一致
3. 测试前端功能是否能正常与后端交互
4. 验证同步、详情查看、执行触发等功能是否正常工作

## 预期成果

通过本次检查和调整，确保前端 `/kanban/index` 页面的所有功能都能与后端 API 接口正确对应，系统能够稳定运行，用户体验良好。
