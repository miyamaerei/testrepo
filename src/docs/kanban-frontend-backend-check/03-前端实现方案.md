## 实现目标

本方案旨在确保前端 `/kanban/index` 页面的所有功能都能与后端 API 接口正确对应，主要包括：

- 验证前端 API 调用是否与后端接口匹配
- 确保前端数据结构转换逻辑正确，特别是处理字段大小写问题
- 优化前端与后端的交互方式
- 确保前端功能的稳定性和可靠性

### 2. 数据结构转换

### 3. 同步功能

前端在 `onSyncAzure` 方法中调用 `triggerSyncFromAzureBoards` 方法触发 Azure Boards 同步：

```typescript
const onSyncAzure = async () => {
  try {
    const res = await api.triggerSyncFromAzureBoards();
    if (res && res.status) {
      ElMessage.success(res.message || '同步已触发');
      await loadKanbanData();
    } else {
      ElMessage.error(res?.message || '同步失败');
    }
  } catch (e) {
    ElMessage.error('同步失败');
    console.error(e);
  }
};
```
