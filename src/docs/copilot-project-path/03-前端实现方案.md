### 4. 看板页面修改

修改看板页面，确保加载卡片时包含项目信息：

**文件**: `src/frontend/vol.web/src/views/Kanban/Index.vue`

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue';
import KanbanCard from '@/components/Kanban/KanbanCard.vue';
import { getKanbanCards } from '@/api/kanban';
import type { ExecutionCardDetail, KanbanData, ExecutionCardStatus } from '@/types/kanban';

const loading = ref(false);
const kanbanData = ref<KanbanData>({} as KanbanData);

const loadCards = async () => {
  loading.value = true;
  try {
    const response = await getKanbanCards();
    // 按状态分组
    const groupedData: KanbanData = {} as KanbanData;
    Object.values(ExecutionCardStatus).forEach(status => {
      if (typeof status === 'number') {
        groupedData[status] = response.data.filter((card: ExecutionCardDetail) => card.Status === status);
      }
    });
    kanbanData.value = groupedData;
  } catch (error) {
    console.error('Load cards failed:', error);
  } finally {
    loading.value = false;
  }
};

const handleRefresh = () => {
  loadCards();
};

onMounted(() => {
  loadCards();
});
</script>
```

### 5. 错误处理

在前端添加错误处理，显示执行失败的原因：

**文件**: `src/frontend/vol.web/src/api/http.js`

```javascript
// 响应拦截器
axios.interceptors.response.use(
  response => {
    // 现有逻辑...
    return response;
  },
  error => {
    // 增强错误处理
    if (error.response) {
      const message = error.response.data?.message || '请求失败';
      
      // 特别处理项目路径相关错误
      if (message.includes('Working directory')) {
        ElMessage.error('项目路径不存在或不可访问，请检查项目配置');
      } else {
        ElMessage.error(message);
      }
    } else {
      ElMessage.error('网络错误，请检查网络连接');
    }
    return Promise.reject(error);
  }
);
```

**说明**：将错误处理放在 `http.js` 中的原因是：

1. **集中处理**：在 axios 拦截器中处理错误可以实现集中化的错误处理逻辑，确保所有 API 请求的错误都能得到一致的处理
2. **代码复用**：避免在每个 API 调用中重复编写相同的错误处理逻辑
3. **全局覆盖**：确保所有 API 请求的错误都能被捕获和处理，包括网络错误、超时等

同时，在特定的 API 调用中，也可以添加额外的错误处理逻辑：

**文件**: `src/frontend/vol.web/src/api/kanban.ts`
