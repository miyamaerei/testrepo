# Copilot 项目路径关联 - 后端实现方案

## 一、后端项目结构分析

通过分析现有后端项目，我们了解到：

1. **核心服务**：
   - `AiExecutionService`：负责 AI 执行的核心服务
   - `CopilotCliClient`：封装 GitHub Copilot CLI 的调用
   - `IExecutionCardRepository`：卡片数据访问接口
   - `IProjectRepositoryRepository`：项目仓库数据访问接口

2. **数据模型**：
   - `ExecutionCard`：执行卡片实体，包含项目关联信息
   - `ProjectRepository`：项目仓库实体，包含本地目录路径

3. **配置管理**：
   - `appsettings.json` 中的 `CopilotCli` 配置节

### 2. 修改 AiExecutionService

修改 `AiExecutionService.cs`，在执行前获取卡片绑定的项目信息：

```csharp
public async Task ExecuteAsync(int cardId)
{
    // 获取卡片信息，包含项目关联
    var card = await _executionCardRepositories.FindFirstAsync(
        p => p.Id == cardId,
        p => p.Include(q => q.ProjectRepository) // 加载项目关联
    );
    
    if (card == null)
    {
        _logger.LogError("Card not found: {CardId}", cardId);
        throw new Exception($"Card not found: {cardId}");
    }
    
    // 确定工作目录
    string workingDirectory = GetWorkingDirectory(card);
    
    // 验证工作目录
    if (!Directory.Exists(workingDirectory))
    {
        _logger.LogError("Working directory does not exist: {WorkingDirectory}", workingDirectory);
        throw new Exception($"Working directory does not exist: {workingDirectory}");
    }
    
    // 构建执行 prompt
    var prompt = BuildExecutionPrompt(card);
    
    // 执行 via Copilot CLI，传递工作目录
    var output = await _copilotCli.ExecutePromptAsync(prompt, workingDirectory);
    
    // 提交结果
    var result = await _submitService.SubmitExecutionResultAsync(
        card.Id,
        (int)ExecutorType.AI,
        "GitHub Copilot CLI",
        output,
        output
    );
    
    _logger.LogInformation(
        "AI execution completed for card {CardId}, Spec passed: {IsPassed}, new status: {NewStatus}",
        card.Id, result.IsSpecPassed, result.NewStatus
    );
}

private string GetWorkingDirectory(ExecutionCard card)
    {
        // 如果卡片绑定了项目，使用项目的本地工作目录
        if (card.ProjectRepositoryId.HasValue && card.ProjectRepository != null && !string.IsNullOrEmpty(card.ProjectRepository.LocalWorkingDir))
        {
            return card.ProjectRepository.LocalWorkingDir;
        }
        
        // 否则使用全局配置的工作目录
        return _configuration.GetValue<string>("CopilotCli:WorkingDirectory", "C:\\repo");
    }
```

### 4. 错误处理增强

在 `AiExecutionService` 中添加更详细的错误处理：

```csharp
try
{
    // 执行逻辑...
}
catch (Exception ex)
{
    _logger.LogError(ex, "AI execution failed for card {CardId}", cardId);
    
    // 获取最新卡片状态
    var freshCard = await _executionCardRepositories.FindFirstAsync(p => p.Id == cardId);
    if (freshCard != null)
    {
        // 处理执行失败
        freshCard.Status = ExecutionCardStatus.Failed;
        freshCard.FailureCount++;
        freshCard.LastFailedAt = DateTime.UtcNow;
        freshCard.FailureReason = ex.Message;
        
        _executionCardRepositories.Update(freshCard);
        await _executionCardRepositories.SaveChangesAsync();
    }
    
    throw;
}
```

## 四、技术要点

1. **路径处理**：确保不同操作系统下的路径格式正确
2. **权限管理**：确保 Copilot 进程有足够权限访问项目目录
3. **异常处理**：提供清晰的错误信息，便于问题排查
4. **日志记录**：详细记录执行过程和错误信息
5. **性能优化**：避免重复的数据库查询和文件系统操作

## 五、测试场景

1. **正常场景**：卡片绑定项目，项目路径存在且可访问
2. **路径不存在**：卡片绑定项目，但项目路径不存在
3. **无项目绑定**：卡片未绑定项目，使用默认工作目录
4. **路径不可访问**：项目路径存在但权限不足
