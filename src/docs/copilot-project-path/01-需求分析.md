# Copilot 项目路径关联 - 需求分析

## 一、核心需求

1. **项目路径关联**：当卡片执行时，Copilot 的工作目录应与卡片绑定的项目路径关联
2. **动态工作目录**：根据卡片绑定的项目信息，动态设置 Copilot 执行的工作目录
3. **执行环境一致性**：确保 Copilot 执行时能够访问到正确的项目代码和资源
4. **路径验证**：在执行前验证项目路径的存在性和可访问性
5. **错误处理**：当项目路径不存在或不可访问时，提供明确的错误提示

## 二、技术实现要点

1. **数据模型关联**：
   - 确认卡片与项目的关联关系（ExecutionCard 与 ProjectRepository 的关联）
   - 确保项目表中存储了完整的本地目录路径

2. **执行服务修改**：
   - 修改 AiExecutionService，在执行前获取卡片绑定的项目信息
   - 根据项目信息构建正确的工作目录路径
   - 将工作目录传递给 CopilotCliClient

3. **配置管理**：
   - 保留全局 Copilot 配置作为默认值
   - 当卡片绑定项目时，优先使用项目的本地目录作为工作目录

4. **错误处理机制**：
   - 当项目路径不存在时，记录错误并通知用户
   - 当项目路径存在但不可访问时，提供明确的错误信息

5. **测试验证**：
   - 验证不同项目绑定的卡片执行时使用正确的工作目录
   - 验证项目路径不存在时的错误处理
   - 验证执行结果与项目的一致性

## 三、需求优先级

| 需求项 | 优先级 | 说明 |
|--------|--------|------|
| 项目路径关联 | 高 | 核心功能，必须实现 |
| 动态工作目录 | 高 | 确保执行环境正确 |
| 执行环境一致性 | 高 | 保证执行结果的有效性 |
| 路径验证 | 中 | 提升系统稳定性 |
| 错误处理 | 中 | 提升用户体验 |

## 四、业务场景

1. **多项目环境**：系统中存在多个项目，每个卡片绑定到不同的项目
2. **跨项目执行**：同一用户可能需要在不同项目间切换执行任务
3. **项目路径变更**：项目本地目录可能发生变更，系统需要能够适应

## 五、技术挑战

1. **路径解析**：确保不同操作系统下的路径格式正确
2. **权限管理**：确保 Copilot 进程有足够权限访问项目目录
3. **环境变量**：确保 Copilot 执行时能够正确加载项目相关的环境变量
4. **错误恢复**：当项目路径变更时，如何处理正在执行的任务
