## ✅ 每批次完成后的验证步骤

每完成一个批次，都要做以下验证：

```
完成批次开发
    ↓
静态代码检查（用 Copilot 检查）
    ↓
编译项目，确保能通过编译
    ↓
单元测试（如果写了单元测试）
    ↓
手动验证功能
    ↓
验证通过 → 下一个批次
验证不通过 → 修复问题，重新验证
```

## ⚙️ GitHub Copilot CLI 编程模式执行建议

因为你使用的是 GitHub Copilot CLI，可以用编程模式批量处理：

```bash
# 每次处理一个任务，完成后再继续下一个
# 示例：创建第一个实体
copilot -p "
我正在开发 E-Kanban 项目，基于 Vol.Core 框架，请帮我创建 BoardWorkItem 实体。
开发规范：
$(cat docs/volcore-development-guide.md)
需求：
$(cat docs/requirements-structured.md)
现有实体结构参考：
$(cat VOL.Entity/SystemModels/User.cs)

请帮我生成代码。
" --allow-all-tools
```

**原则**：一次只做一个任务，做完验证了再做下一个，这就是小步快跑。
