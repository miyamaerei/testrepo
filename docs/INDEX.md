# E-Kanban 开发知识索引

> **快速导航**：本文档是 E-Kanban 项目的知识索引，帮助 AI 和开发者快速定位所需信息。

---

## 🎯 我要做什么？

### 场景 1：接到新需求，开始开发
```
步骤 1：理解需求 → 打开 kb-requirements.md 或 src/feature-records/需求名称/
步骤 2：定位代码 → 打开 kb-module-index.md，搜索功能关键词
步骤 3：参考规范 → 打开 kb-code-patterns.md，找到对应的代码模板
步骤 4：开始开发 → 按照 guide-development.md 的流程执行
```

### 场景 2：开发过程中遇到问题
```
步骤 1：查问题库 → 打开 kb-problem-solutions.md，搜索关键词
步骤 2：查相似模块 → 打开 kb-module-index.md，找到相似功能的代码
步骤 3：查代码模式 → 打开 kb-code-patterns.md，参考代码模板
```

### 场景 3：了解项目整体结构
```
步骤 1：看代码资产 → 打开 kb-code-assets.md
步骤 2：看项目结构 → 打开 kb-module-index.md 的第 1 章
```

### 场景 4：查看某个需求的详细信息
```
步骤 1：看需求列表 → 打开 src/feature-records/
步骤 2：进入具体需求文件夹 → 查看 01-需求分析.md、02-后端实现方案.md 等
```

## 📚 核心知识库详解

### 1. 模块索引（快速定位）

**文档**：[kb-module-index.md](./kb-module-index.md)

**用途**：查找每个功能对应的代码位置

#### 📖 文档结构

```
kb-module-index.md
├── 1. 项目整体结构           # 项目目录树，了解代码应该放在哪里
├── 2. 需求-代码映射表        # 核心部分：功能点 → 文件位置映射
│   ├── 领域模型              # 实体类
│   ├── 仓储层                # 仓储接口和实现
│   ├── 服务层                # 服务接口和实现
│   ├── Spec Engine           # Spec 相关
│   ├── GitHub Copilot CLI    # AI 执行相关
│   ├── 定时任务              # Quartz 任务
│   ├── API 控制器            # Web API
│   ├── 配置文件              # appsettings.json
│   └── 前端 UI               # Vue 组件和页面
├── 3. 技术选型               # 后端、前端、数据库技术栈
├── 4. GitHub Copilot CLI 集成 # AI 执行机制说明
├── 5. 开发顺序建议           # 按依赖关系排序的开发顺序
└── 6. 结论                   # 总结
```

#### 🔍 如何使用本文档

**场景 A：新需求开发，不知道代码放哪里**
```
1. 打开文档，跳到第 2 章"需求-代码映射表"
2. 在表格中搜索相似功能的名称
3. 查看该功能的"需要新增/修改的文件"列
4. 参考相似功能的文件位置，确定新代码的位置
```

**场景 B：查找某个功能的实现位置**
```
1. 打开文档，跳到第 2 章"需求-代码映射表"
2. 在表格中搜索功能名称（如 "ExecutionCard"）
3. 查看"需要新增/修改的文件"列，找到文件路径
4. 在 IDE 中打开对应文件
```

**场景 C：了解项目整体结构**
```
1. 打开文档，跳到第 1 章"项目整体结构"
2. 查看目录树，了解各个模块的位置
3. 理解分层架构：Entity → Repository → Service → Controller → Frontend
```

#### 💡 使用技巧

- **搜索关键词**：使用 Ctrl+F 搜索功能名称（如 "ExecutionCard"、"Sync"）
- **参考相似功能**：找到相似功能的代码位置，参考其实现方式
- **注意依赖关系**：查看"依赖框架模块"列，了解模块之间的依赖

## 🚀 快速开始

### 对于 AI 助手

当接到开发任务时，按以下顺序查找信息：

```
步骤 1：理解需求
  → 打开 kb-requirements.md 或 src/feature-records/需求名称/
  → 阅读需求分析文档

步骤 2：定位代码
  → 打开 kb-module-index.md
  → 搜索相似功能的代码位置
  → 确定新代码应该放在哪里

步骤 3：参考规范
  → 打开 kb-code-patterns.md
  → 找到对应的代码模板
  → 复制模板并根据需求修改

步骤 4：查看相似模块（重要！）
  → 在 kb-module-index.md 中找到相似功能
  → 阅读相似模块的代码实现
  → 复用相似的模式和逻辑

步骤 5：遇到问题
  → 打开 kb-problem-solutions.md
  → 搜索关键词，查找解决方案

步骤 6：开始开发
  → 按照 guide-development.md 的流程执行
```

### 对于人类开发者

```
1. 新人入门：先看 kb-code-assets.md 了解项目结构
2. 开始开发：看 guide-development.md 了解流程
3. 查代码位置：看 kb-module-index.md
4. 参考代码模板：看 kb-code-patterns.md
5. 遇到问题：看 kb-problem-solutions.md
```

| 文档 | 更新内容 | 更新时机 |
